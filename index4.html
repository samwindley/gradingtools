Custom Student Grading Tools

Letting student preferences decide grading methods, building the tools they need to grade each other, and analyzing the results.

Step 1

Giving students a voice when creating teams

In lieu of a final exam, the second-year course (Practical English IV) would conclude with a "team presentation".

Before creating teams, we used a tool called Clever Groups to collect students' preferences about who they would like to work with.

When students followed the Clever Groups link we gave them, they were asked to select their first name (and an initial where necessary to distinguish between students with the same first name) from a list of students in the class.

After a student selects their name, they see a dropdown menu containing the names of all their classmates and they are asked to choose their first preference (the student they most want to work with). Students can select as many preferences as they like (even zero).

After all students have had an opportunity to enter their preferences, we log in to Clever Groups and ask the site to generate teams given certain parameters (maximum team size, number of teams, etc.). Clever Groups then uses an algorithm to identify the optimum allocation of students to teams (i.e., the allocation that best fits student preferences).

We could even have triggered this team allocation in front of the students, during class, to demonstrate that we were not adjusting the allocation to suit our personal preferences, etc.

Step 2

Collecting student grading preferences

The next step was to ask students questions about their presentation grading preferences, such as which aspects of a presentation they think should affect their grade, and who they trust to give them a fair grade.

Students were already familiar with Kahoot! from other classroom activities so we used a Kahoot! quiz (polls and scales) to pose the questions. We asked students about the importance of the following:

  • preparation
  • slides
  • speaking loudly and clearly
  • presentation length
  • being entertaining
  • choosing an interesting topic
  • how cohesive the team's presentation is overall
  • grading other students fairly

We also asked students about how they wanted to grade other students. Specifically, we asked them about:

  • mechanics: online or on-paper
  • timing: during the presentations or afterwards
  • composition: one score for the whole team or separate scores per speaker
  • scaffolding: whether they wanted us to provide a rubric or not

Finally, we asked students about how much they trusted the following people to give them a fair grade:

  • The teacher
  • Their teammates
  • Their classmates

Step 3

Combine Views Using Stacks

Beyond the title view you created in the previous section, you'll add text views to contain details about the landmark, such as the name of the park and state it's in.

When creating a SwiftUI view, you describe its content, layout, and behavior in the view's body property. However, the body property only returns a single view. You can combine and embed multiple views in stacks, which group views together horizontally, vertically, or back-to-front.

VStack(alignment: .leading) { Text("Turtle Rock") .font(.title) HStack { Text("Joshua Tree National Park") .font(.subheadline) Spacer() Text("California") .font(.subheadline) } }

VStack arranges views vertically, while HStack arranges them horizontally. The Spacer pushes views apart to use available space.

Notice how the preview on the right now shows the actual code implementation instead of a mockup. This is useful when you want to highlight specific code snippets that relate to the section.

Section 4

Step 4

Create a Custom Image View

With the name and place views all set, the next step is to add an image for the landmark.

Instead of adding more code to this file, you'll create a custom view that applies a mask, border, and drop shadow to the image.

Image("turtlerock") .clipShape(Circle()) .overlay(Circle().stroke(Color.white, lineWidth: 4)) .shadow(radius: 7)

The clipShape modifier masks the image to a circle. The overlay adds a white border, and shadow gives it depth.

These modifiers can be combined to create sophisticated visual effects with just a few lines of code.

Section 5

Step 5

Use SwiftUI Views From Other Frameworks

Next you'll add a map to the detail view. You can use the Map view from MapKit to render a map interface.

When you build views in SwiftUI, you can use standard views and views from other frameworks side-by-side. In this case, you'll include a Map view from MapKit along with your custom SwiftUI views.

Map(coordinateRegion: $region) .frame(height: 300)

The Map view integrates seamlessly with SwiftUI. The $ prefix creates a binding, allowing the map to update its displayed region.

Notice how the preview on the right changes as you scroll through different sections. Each section has its own corresponding preview.

Conclusion

You've now created a landmark detail view that combines text, images, and maps. The split-screen layout allows you to see your changes in real-time as you follow along with the tutorial.

struct ContentView: View {
    var body: some View {
        VStack(alignment: .leading) {
            Text("Turtle Rock")
                .font(.title)
            HStack {
                Text("Joshua Tree")
                    .font(.subheadline)
                Spacer()
                Text("California")
                    .font(.subheadline)
            }
        }
    }
}
